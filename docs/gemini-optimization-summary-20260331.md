# Gemini 模型优化与计费管理总结报告 (2026-03-31)

## 1. 核心问题背景
用户反馈其 $10 预付款被持续扣除，而非优先使用 $300 赠送金额。经分析，主要原因是应用在对话和代码优化过程中频繁触发后台任务（如自动生成标题、智能建议等），这些任务默认调用了 Standard Flash 或 Pro 模型，导致额度被快速消耗。

## 2. 已实施的优化方案 

### A. 模型套装化 (Model Suites)
为了简化 UI 并统一逻辑，我们将零散的模型选择重构为三个“套装”：
- **Pro 套装 (最强逻辑/顶级生图)**: 文本 `gemini-3.1-pro-preview` + 生图 `gemini-3-pro-image-preview`。
- **Flash 套装 (全能/极速生图)**: 文本 `gemini-3-flash-preview` + 生图 `gemini-3.1-flash-image-preview` (原 Banana 2)。
- **Lite 套装 (日常/经典生图)**: 文本 `gemini-3.1-flash-lite` + 生图 `gemini-2.5-flash-image` (原 Banana 经典)。

### B. 后台任务硬编码 (Cost-Saving Guardrails)
为了彻底杜绝“莫名其妙扣费”，我们将所有非用户直接触发的后台任务强制绑定到 **Gemini 3.1 Flash Lite**：
- **自动生成会话标题**
- **智能建议按钮 (Smart Suggestions)**
- **视觉搜索与物体识别 (Visual Search)**
- **图层分析与坐标检测 (Layer Analysis)**
- **结果：** 这些任务现在消耗的是 Lite 模型的免费/极高配额（每天 15 万次），不再占用 Pro (250次) 或 Standard Flash (1000次) 的珍贵配额。

### C. 流程策划 (Workflow Planning) 智商升级
- **逻辑调整：** 将“流程策划 (planGraph)”从后台任务中提出来，使其跟随用户选择的套装。
- **效果：** 当用户选择 Pro 套装时，系统会动用 Pro 模型来规划 AI 节点，逻辑更严密；同时支持“思考等级”设置，开启 MEDIUM (HIGH) 后可进行深度推理。

### D. 思考等级 (Thinking Level) 逻辑修正
- **文本/规划：** 严格遵循 UI 上的 LOW/MEDIUM 开关。
- **生图/修图：** **自动/托管模式**。移除了生图任务中的显式思考等级参数，防止 API 报错（如 Banana 2 在精准编辑时拒绝 LOW 等级导致的 400 错误）。

## 3. UI 与体验改进
- **模型重命名：** 将加号菜单中的模型名称从内部代号（Banana 系列）改为官方名称（Gemini 3.1 Pro/Flash），消除用户困惑。
- **默认设置：** 默认选择 Flash 套装，兼顾速度与成本。

## 4. 计费逻辑最终确认
- **日常闲聊/简单指令：** 选 Flash 套装，每天 10,000 次，极速响应。
- **高难度逻辑/复杂规划：** 选 Pro 套装，每天 250 次，好钢用在刀刃上。
- **后台自动任务：** 走 Flash Lite 通道，每天 150,000 次，完全不用担心。

## 5. 最终计费逻辑全景图

| 任务类型 | 模型来源 | 思考等级 (Thinking Level) | 建议套装 |
| :--- | :--- | :--- | :--- |
| **日常聊天** | 套装-Text | **受控 (LOW/MED)** | Flash / Lite |
| **流程策划** | 套装-Text | **受控 (LOW/MED)** | Pro / Flash |
| **生图/修图** | 套装-Image | **托管 (Auto)** | 任意 |
| **后台分析** | **Lite (固定)** | **固定 LOW** | 无需担心 |
| **生成视频** | **Veo (固定)** | **托管 (Auto)** | 独立功能 |
