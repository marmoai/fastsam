# 语义图层分析功能工作流程详解

本文档详细拆解了“语义图层分析”功能从宏观架构到微观代码的完整运作流程。整个流程分为三大阶段：**触发与分析阶段**、**裁剪与重建阶段**、**抠图与后处理阶段**。

## 阶段一：触发与语义分析 (识别图片里有什么)


**用户操作：** 上传一张图片，点击“语义图层分析”按钮。

**工作逻辑：**
1. **前端触发：** 用户点击按钮后，UI 层面（`src/ui/modals.js`）捕获点击事件，获取当前图片的 Base64 数据。
2. **调用分析函数：** 前端调用 `src/ai-services/skills-engine.js` 中的 `analyzeImageLayers(file)` 函数。
3. **AI 模型调用 (核心)：**
   * **操作者：** 后端/云端 AI 服务。
   * **调用工具/模型：** **Gemini API**（通常是 Gemini 1.5 Pro 或 2.5 Pro 等多模态大模型）。
   * **执行动作：** 代码向 Gemini 发送原始图片，并附带一段极其严格的 Prompt（提示词）。
   * **Prompt 核心要求：** 要求 Gemini 扮演“图层场景重建系统”，将图片拆解为 3-8 个逻辑图层（如前景资产、背景板）。要求返回严格的 JSON 格式，包含每个物体的：
     * `id` 和 `name`（名称）
     * `layerType`（图层类型：前景或背景）
     * `semanticType`（语义标签：如沙发、桌子、墙壁）
     * **`bbox`（边界框坐标：[ymin, xmin, ymax, xmax]，这是后续裁剪的关键）**
4. **前端渲染列表：** 前端接收到 Gemini 返回的 JSON 数组后，将其解析并在 UI 的“图层列表”中渲染出来。此时，用户看到了图片被识别出的各个元素。

---

## 阶段二：提取独立图层 - 裁剪与重建 (把物体单独拿出来)

**用户操作：** 在图层列表里点击其中一个图层（例如“沙发”），选择“提取图层”。

**工作逻辑：**
这一步主要由 `src/ui/layer-manager.js` 中的 `triggerLayerExplosion` 函数负责。

1. **前端精准裁剪 (Crop)：**
   * **操作者：** 浏览器前端。
   * **调用工具：** **HTML5 Canvas API**。
   * **执行动作：** 根据第一阶段 Gemini 返回的该物体的 `bbox`（边界框坐标），前端在内存中创建一个新的 Canvas，利用 `ctx.drawImage` 将原图中该坐标范围内的像素“裁剪”下来。为了防止边缘被切断，代码还向外扩展了 1% 的 padding（内边距）。
   * **产出物：** 一张只包含该物体及其周围一小部分复杂背景的局部截图（`croppedImage`）。

2. **AI 资产重建 (去除复杂背景，替换为纯色背景)：**
   * **操作者：** 后端/云端 AI 服务。
   * **调用工具/模型：** **Gemini API**（调用 `editOrQueryImageWithGemini` 函数）。
   * **执行动作：** 前端将刚才裁剪下来的局部截图发给 Gemini，并附带一段 Prompt。
   * **Prompt 核心要求：** “精准提取图像工具。提取物体并将其放置在纯色背景上。零幻觉（不改变物体本身），只改变背景。” *(注：最佳实践是直接要求替换为 `#00FF00` 纯绿背景，以优化后续抠图)*。
   * **产出物：** 一张物体保持原样，但周围复杂背景被替换为纯色（如纯白或纯绿）的图片（`reconstructedDataUrl`）。

---

## 阶段三：抠图与后处理 (变成透明背景的 PNG)

接下来的逻辑依然在 `triggerLayerExplosion` 函数中继续执行。

1. **前端垫底色 (可选/待优化)：**
   * **操作者：** 浏览器前端。
   * **调用工具：** **HTML5 Canvas API**。
   * **执行动作：** 前端创建一个绿色的 Canvas (`#00FF00`)，然后把阶段二 Gemini 返回的图片画上去。
   * *(注：如果阶段二 Gemini 返回的图片已经是不透明的纯色背景，这一步的垫底色会被完全遮挡，属于无效操作。优化方案是直接在阶段二让 Gemini 返回绿色背景，跳过此步)*。

2. **AI 自动抠图 (移除纯色背景)：**
   * **操作者：** 浏览器前端（本地计算）。
   * **调用工具/模型：** **`@imgly/background-removal`**（这是一个基于 WebAssembly 和本地轻量级 AI 模型的开源前端抠图库）。
   * **执行动作：** 前端将上一步的图片传给这个库的 `removeBackground` 函数。该库的 AI 模型会识别出背景，并将其 Alpha 通道设为 0（透明）。
   * **产出物：** 一张背景透明的 Blob 数据。

3. **前端边缘收缩 (消除残留边缘)：**
   * **操作者：** 浏览器前端。
   * **调用工具：** **HTML5 Canvas API (像素级操作 `getImageData`)**。
   * **执行动作：** 为了防止抠图后物体边缘残留背景色（如白边或绿边），代码遍历了图片的每一个像素。如果发现某个像素是半透明的（Alpha 值在 1 到 250 之间），代码会强制将其 Alpha 值进一步降低（乘以 0.5 或直接设为 0），从而实现“边缘收缩”（Edge Shrinking）或“羽化”效果。

4. **保存与更新 UI：**
   * **操作者：** 浏览器前端。
   * **执行动作：** 将最终处理好的透明 PNG 图片保存为应用内的独立图层资产，并更新画布，用户此时就可以自由拖拽这个独立的透明图层了。

---

## 总结：核心工具链一览表

| 步骤 | 核心任务 | 执行者 | 使用的工具 / AI 模型 |
| :--- | :--- | :--- | :--- |
| **1** | 识别图片中的物体及坐标 | 云端 AI | **Gemini API** (多模态视觉分析) |
| **2** | 根据坐标裁剪局部图片 | 浏览器前端 | **HTML5 Canvas** (`drawImage`) |
| **3** | 将局部复杂背景替换为纯色 | 云端 AI | **Gemini API** (图像编辑/重建) |
| **4** | 尝试垫底色 (视情况可省略) | 浏览器前端 | **HTML5 Canvas** (`fillRect`) |
| **5** | 将纯色背景移除，变为透明 | 浏览器前端 | **`@imgly/background-removal`** (本地 AI 抠图模型) |
| **6** | 像素级边缘收缩，去残留边 | 浏览器前端 | **HTML5 Canvas** (`getImageData` 像素遍历) |

> **架构亮点：** 整个流程巧妙地结合了**云端大模型**（Gemini 负责高智商的语义理解和复杂背景去除）、**本地小模型**（@imgly 负责快速的纯色背景透明化）以及**前端原生 API**（Canvas 负责精准的裁剪和像素修补）。
