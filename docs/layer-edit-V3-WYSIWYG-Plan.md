# 语义图层“所见即所得 (WYSIWYG)”自由构图流程规划

## 一、 现状痛点与重构目标

### 1. 当前流程分析
目前系统在处理图层移动和融合时（如 `handleQuickFusionSync` 和 `handleRecomposeVerification`），采用的是**“严格坐标逆向映射”**逻辑：
*   **拖拽与预览**：用户拖动图层时，系统会更新 `bbox` 并在底板上显示一个 BBOX 框作为位置预览。
*   **坐标映射**：合成时，系统计算图层当前的偏移量（`relDx`, `relDy`），然后逆向推导回原图的 `originalBbox` 坐标系。
*   **回贴验证**：点击“回贴验证”后，系统根据映射坐标生成蒙版或粗糙合成图，再交给大模型处理。

### 2. 重构目标
*   **取消逆向映射**：彻底废弃基于 `originalBbox` 和偏移量的复杂计算。
*   **所见即所得**：用户在底板上直接拖拽、缩放、旋转图层实体（`.canvas-layer`），屏幕上看到的样子，就是最终合成草图的样子。
*   **取消 BBOX 预览包袱**：既然是所见即所得，拖动时图层本身就是预览，不再需要额外的 BBOX 框线跟随和位置预览提示。
*   **保留闭环逻辑**：保留点击“回贴验证”提交草图给大模型出图的核心业务流。

## 二、 核心流程改造规划

### 阶段一：交互层改造（取消 BBOX 预览，实现纯视觉拖拽）
**目标文件**：`src/ui/workbench/layers.js`
1.  **移除 BBOX 边框与占位提示**：
    *   在渲染 `.canvas-layer` 时，移除不必要的边框（如 `box-shadow` 模拟的 BBOX 框），让图层（`cutoutUrl` 图像）本身成为唯一的视觉主体。
2.  **升级拖拽与变换逻辑**：
    *   拖动时，直接更新图层的 `left` 和 `top`（百分比）。
    *   支持通过 CSS `transform` 直接对图层进行缩放（Scale）和旋转（Rotate）。
    *   **关键点**：拖动结束时，不再需要强制换算回 0-1000 的严格 `bbox` 数组，或者即使保留 `bbox` 格式，也仅作为当前百分比的记录，不再用于逆向映射。

### 阶段二：合成引擎重构（从“坐标映射”走向“正向渲染”）
**目标文件**：`src/ui/modals.js` (重写 `handleRecomposeVerification` 函数) 和 `src/ui/layer-manager.js`
1.  **废除旧的映射计算**：
    *   删除 `relDx`, `relDy`, `spawnLeft`, `spawnTop` 等相对位移计算代码。
2.  **构建 WYSIWYG 草图渲染器**：
    *   **创建离屏 Canvas**：尺寸与底板（Clean Plate）的真实物理像素一致。
    *   **绘制底板**：将底板图片 1:1 铺满 Canvas。
    *   **正向绘制图层**：遍历底板上的所有可见图层（`.canvas-layer`）。读取它们当前的 DOM 属性：
        *   `style.left` / `style.top` -> 转换为 Canvas 上的 X/Y 像素坐标。
        *   `style.width` / `style.height` -> 转换为 Canvas 上的宽高像素。
        *   `style.transform` -> 转换为 Canvas 的 `ctx.rotate` 和 `ctx.scale`。
    *   将图层的 `cutoutUrl`（抠图素材）按照上述计算好的真实像素位置和形变，直接 `ctx.drawImage` 到 Canvas 上。
3.  **输出直观草图**：将 Canvas 导出为 `sketchDataUrl`。这张图将与用户在屏幕上看到的排版**完全一致**。

### 阶段三：大模型深度融合（AI Fusion）
**目标文件**：`src/ui/modals.js`
1.  **无缝衔接**：将阶段二生成的 `sketchDataUrl` 直接作为输入图片。
2.  **更新 Prompt 策略**：
    *   **旧 Prompt**：偏向于“在蒙版区域内重绘”。
    *   **新 Prompt**：“这是一张用户自由排版的构图草图（Composition Sketch）。请将其转化为一张高度逼真、光影和谐的最终图像。必须严格遵循草图中物体的空间位置、大小比例、层级遮挡关系。你需要统一全局的光照、阴影、透视关系，并完美融合物体边缘。”
3.  **结果闭环**：大模型返回融合后的高清图，作为新的工作台项目展示，完成“回贴验证”的最终目的。

---

## 三、 总结与预期收益
通过这次重构，我们将前端的职责从**“复杂的坐标数学计算器”**转变为**“忠实的屏幕截图器”**。
1.  **代码大幅精简**：去掉了极易出错的相对坐标映射逻辑。
2.  **用户体验飞跃**：真正的自由构图，用户可以随意放大、缩小、旋转图层，甚至把图层拖到原 BBOX 范围之外，所见即所得。
3.  **视觉干扰减少**：去掉了拖动时的 BBOX 框线，让用户的注意力集中在图像内容本身。
