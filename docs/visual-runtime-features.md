# Visual Asset Runtime - 系统架构与战略护城河

本系统不仅是一个视觉创作工具，更是一个记录、理解并预测“人类视觉创作决策过程”的**上层操作系统 (Operating System for Visual Creation)**。

---

## 1. 系统核心架构：决策驱动的 Runtime

与传统生成式 AI 直接输出结果不同，本系统引入了一个严密的 Runtime 环境：

*   **Dispatcher (调度核心):** 系统所有操作的单一入口。拦截所有变动，确保意图 (`Intent`) 能够精确映射到最终的 `workbenchState` (图层状态)。
*   **GraphEngine (决策图谱):** 将每一个创作场景抽象为语义图谱。不是简单存图，而是记录物体之间的逻辑关系（耳机主体、背景、文字）。
*   **CreativeMemoryLayer (决策日志):** 系统最关键的资产库。它不记录参数修改，而是记录“人类为什么做出这个决定”。

---

## 2. 数据飞轮：从工具到智能

核心价值在于：我们正在构建一个**“人类视觉决策资产库”**。

### 飞轮原理：
1.  **记录 (Capture):** 每次用户触发 `Dispatcher`，系统打包记录：`{ Context(场景: 汉堡) -> Intent(意图: 提升食欲) -> Delta(参数偏置: +暖色) }`。
2.  **提纯 (Refining):** 去除噪点，建立“意图”与“执行结果”的干净对应关系。
3.  **反制 (Reverse-Control):** 数据不再静止。经过训练的 `SemanticRecommender` 根据用户的历史路径，在下一次面对类似 Context 时，直接**推荐决策路径**（而非仅仅推荐参数）。

---

## 3. 我们的护城河：为什么在 AI 洪流中能活下来？

OpenAI/Google 拥有更强的模型，但这无法替代我们。我们的护城河在于：

*   **数据资产独特性:** 世界上没有任何一家公司拥有结构化、带有意图标记的“连续创作行为日志”。
*   **决策反馈闭环:** 我们的数据包含用户的“撤销/保留”、“失败路径”和“修正逻辑”。这是人类视觉决策的真实心理映射，而非通过合成生成的通用数据。
*   **Runtime 绑定:** 数据与应用逻辑深度耦合，使得系统具备“自进化”能力，每增加一个用户，系统的推荐能力就指数级提升。

---

## 4. 进化路线图

### 第一阶段 (Current - Instrumentation): 结构化采集
*   确保 `Intent -> Dispatcher -> State` 的对应关系 100% 干净。
*   引入资产演化树 (Evolution Tree) 实现决策路径回溯。

### 第二阶段 (Soon - Prediction): 用户意图推荐
*   `SemanticRecommender` 介入。
*   工作流：导入图片 -> 系统通过 Semantic Matching 弹出 Fusion Panel -> 推荐 2-3 个最优决策路径。用户：从“操作者”变为“审核决策者”。

### 第三阶段 (Strategic Automation): 决策链自动接管
*   AI 完成全场景优化，用户仅决定设计策略。
*   构建全局汇聚的“视觉智慧特征库”，为每一位创作者赋能千人千面的设计决策支持。

## 5. 附：核心功能特性 (Legacy Reference)

(以下为当前系统基础功能的定义，它们是飞轮持续运转的底座)

*   **Intent Capture:** 意图语义化（不仅记录动作，更记录目的）。
*   **Design Patterns:** 一键式设计经验封装。
*   **Graph Intelligence:** 基于语义上下文的智能排序。
*   **Evolution Tree:** 可穿越的决策记录谱系。

> **核心结论：** 我们不生产图像，我们存储“视觉决策的智慧路径”。
